Mobile Growth

it has grown very quickly since 2008 and now 50% of traffic for many industries come from mobile. however mobile use has not come at the expense of desktop use.
we spend about 2.5 hours per day on both desktop and mobile devices. That's the average.. here obviously we spend a lot more time. but notice how it's almost the same amount of time between mobile and desktop

Device Diversity

http://opensignal.com/reports/2014/android-fragmentation/
in 2014 there was about 18,796 (19 thousand) DISTINCT ANDROID DEVICES. the only thing we can really know is the size of the browser window and that's not always the same as the default size because some apps have embedded browsers.. and these browsers can have different sizes.

Web provides more growth for marketing that's from a browser over a native mobile app.

User Behavior

there's a myth called mobile context.. we try to come up with senarios for mobile and desktop while in reality 90% of people start a task on a device then completed on another. how many of you use your mobile device in your office near your laptop because it just works better, or blocked by flo the point is that CUSTOMERS WILL USE MOBILE DEVICES IN EVERY CONTEXT, IF YOU MAKE IT EASY ENOUGH

Unified processes

seperate mobile websites (mobile,tablet,desktop) is dangerouse responsive is better than serparate mobile website you don't want to endup with versions of your website and have to update both. one content, single url, single codebase, same release schedule, one team (called The One Web) many companies like airbnb tried mobile sites and immidiatly moved to rwd.

Responsive Design Values

so there were the reasons karen emphasized for the need for RWD Then ethan talked about the 3 main values of RWD

Thinking "Mobile First"

you gotta think mobile first. this means you design for the small screen first no columns just one stream. then you have to prioritize the content what goes first what is't displayed. this way we define priority not layout. progressive website as an example.

Flexibility in Design

you have to start with a flexible foundation there's adaptive and responsive. adaptive tends to be like a snap like oas. and responsive is more smooth like progressive.

Flexibility in Delivery

THE THREE MOST COMMON WORDS IN A RESPONSIVE PROJECT? MOBILE TABLET DESKTOP THE THREE MOST UNHELPFUL WORDS IN A RESPONSIVE PROJECT? MOBILE TABLET DESKTOP don't develop for devices develop for features on those devices.

Images in RWD

img { max-width: 100%; }
now I want to touch one of the tips Ethan gave that I didn;t know about which was about images. Remember RWD is FLUID GRID, FLEXIBLE IMAGES, MEDIA QUERIES. for images to work in rwd all u need is this rule. the layout will take care of the rest
the challenge with images that we have a bit of a performance problem. the average page size is 2 mb but 61% is images. it's because we have now retina screens.

Compressive Images

compressive images is a way to deliver high quality images in half the size. if you take am image and double its size in photoshop and then export it in low quality you'll get the same picture with smaller size but nearly same quality. it could be very helpful for thumbnails and icons.

Rollout

now how do you rollout RWD? it is a very imoprtant process and you'll need to make some important desicions. there area different stratagy to how to do a rollout and it's different for each org. progressive?

Responsive Retrofit

you recode the front end of your website. capital one did this after their mobile m. site failed and 90% of people were bouncing off of it.

Beta

opposite end of the spectrum is beta. lunching a website in beta means retaining the main site and lunching a m. site is what they start with then make it to tablet then grow up and take over the desktop. Fidelity did that. you let the user opt in and opt out to try the new website. but it adds technical difficulty

Pick A Page

means choosing a page on the website and redsign in it. it seems the safest way for many orgs. microsoft did that and they did it with their home page.. not the best place to start but for Microsoft it worked well because it led to significat visibality in the company and teams and business unites started wanting to use RWD.
